# 多智能体驱动的临床决策系统：Multi-Agent"会诊"脓毒症

> 来源：paperClub 原创，2025年10月10日
> 作者：Victor Iapascurta 等，摩尔多瓦技术大学 + 尼古拉·泰斯特米塔努国立医科大学
> 基金：摩尔多瓦NARD，项目编号25.00208.5007.07/PD

## 核心目标

构建可嵌入早期脓毒症管理流程、无需等待培养结果即可提供个体化抗生素方案并自动校验指南依从性的AI协同系统。

## 系统架构

### 3个专职Agent（CrewAI框架）

1. **脓毒症管理智能体**（Sepsis Management Agent）
   - 分析病例全貌（感染类型、生命体征、合并症）
   - 结合文献生成整体治疗策略（监测频率、支持治疗）

2. **抗生素推荐智能体**（Antibiotic Recommendation Agent）
   - 聚焦药物选择
   - 整合感染部位、本地耐药数据、患者禁忌证
   - 生成个体化抗生素方案

3. **指南依从智能体**（Sepsis Guidelines Compliance Agent）
   - 校验前两个Agent输出是否符合2021版拯救脓毒症运动指南
   - 识别偏差并提出修正建议（启动时间、剂量调整）

### LLM：Palmyra-med-70B

### RAG模块

- **向量库**：Chroma开源向量数据库
- **知识源**：20篇最新脓毒症文献 + 2021版脓毒症管理指南
- **检索流程**：病例信息→提取关键特征→向量检索→LLM生成基于证据的建议

## 验证结果

### 验证病例

MIMIC IV Case ID 10020786：86岁女性，肺炎相关性脓毒症，COPD+房颤+糖尿病+甲减

### 技术评估（TrueLens框架，基于GPT-3.5-turbo）

| 指标 | 得分 |
|------|------|
| 接地性（Groundedness） | 0.82 |
| 答案相关性（Answer Relevance） | 0.79 |
| 上下文相关性（Context Relevance） | 0.78 |

### 临床评估

- 2名专家均判定"临床可接受"
- Cohen's kappa = 0.622（p=0.003），中等一致性
- 差异：专家更关注简洁性/可执行性，LLM更关注证据完整性

## 创新点

1. 首次将多智能体协同应用于脓毒症抗生素决策（管理+药物+指南校验闭环）
2. RAG整合最新文献与指南，解决信息滞后
3. 聚焦CAP相关脓毒症亚型，建议更具针对性

## 局限性

1. 仅1个MIMIC IV病例验证
2. 未对接EMR系统
3. Chroma中本地耐药模式需手动更新

## 未来方向

1. 多中心多病例验证
2. EMR自动对接 + 实时耐药数据接口
3. 预后研究（使用系统 vs 未使用系统的死亡率差异）

## 与重症智算的关联

| 维度 | 他们 | 我们 |
|------|------|------|
| 知识检索 | Chroma向量库（20篇+1指南） | FTS5全文索引（90+篇md） |
| LLM | Palmyra-med-70B | 云端LLM |
| Agent架构 | 3个专职Agent（CrewAI协同） | 单次LLM调用 |
| RAG流程 | 关键词→向量检索→LLM生成 | 停用词剥离→FTS5→LLM归纳 |
| 指南校验 | 独立Compliance Agent | 无 |
| 评估 | TrueLens + 专家Cohen's kappa | 无 |

### 可借鉴

1. 小规模KB（20篇）RAG也有效 → 我们90+篇足够
2. Agent职责拆分（管理/药物/指南校验）对应我们4分类（病理生理/诊断/监测/治疗）
3. RAG接地性0.82证明向量检索+LLM能有效抑制幻觉

### 我们的路径
