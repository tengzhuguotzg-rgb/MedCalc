# v3 审阅回应（修订版）

## 基于实测数据的最终决策

### 指南：不截断

实测数据：430篇指南，中位数2,253字符，91%<5,000字符，99%<10,000字符。
- 不截断=完整喂入LLM，无信息丢失
- 最坏情况1篇指南50,410字符≈33,000 tokens，10篇指南均≤3,000字符≈20,000 tokens
- 在64k上下文下完全可控

### 教材：关键词段落截取（Plan B）

实测数据：51章，中位数19,680字符，最大200,302字符。8章超5万字符。
- 固定截取前N字符：前5,000字符可能只有概述+病理生理，治疗建议全丢
- 需要智能截取：保留与查询相关的段落，丢弃无关段落

## 教材截取策略详解

### 原则

教材章节结构通常为：
```
# 标题
概述段落（1-3段）
## 病理生理
段落...段落...
## 诊断
段落...段落...
## 治疗
段落...段落...
## 并发症
段落...段落...
## 延伸/鉴别
段落...段落...
```

截取目标：**保留标题+概述+与查询关键词相关的段落及其上下文，丢弃无关段落**。

### 算法

```
输入: content(全文), keywords[](来自diag.name + expandedTerms)
输出: excerpted(截取后的文本)

步骤:
1. 将全文按段落拆分（以连续空行\n\n为分隔符）
2. 标记必选段落:
   - paragraphs[0]（标题行，通常是"# 第X章 ..."）
   - paragraphs[1..2]（标题后1-2段，即概述部分）
3. 标记关键词段落:
   - 遍历每个paragraph，检查是否包含keywords中任意一个词
   - 匹配到的段落标记为"命中"
4. 扩展上下文:
   - 每个命中段落，取其前1段+后1段作为上下文
   - 避免重复（多个命中段落可能共享上下文）
5. 按原文顺序合并:
   - 将所有标记段落（必选+命中+上下文）按原始顺序排列
   - 用\n\n连接
6. 长度兜底:
   - 如果截取结果超过 maxExcerptChars(20000字符)，从末尾开始删除非必选段落直到不超限
   - 如果截取结果< 500字符（几乎没命中），回退到取前5000字符
```

### 关键词来源

```
keywords = []
添加: diag.name 分词后的词（去除停用词后）
添加: expandedTerms[]（expandQueryWithLlm的输出）

例: diag.name="重症肺炎"
  refineQuery → "肺炎"（"重症"是停用词被剥离）
  expandedTerms → ["CAP","HAP","VAP","社区获得性肺炎","呼吸机相关"]
  keywords = ["肺炎","CAP","HAP","VAP","社区获得性肺炎","呼吸机相关"]
```

### 上下文窗口：±1段

- 只取命中段落本身：可能信息不完整（如"上述治疗方案..."前文被截断）
- ±1段：保留紧邻的因果/延续关系
- ±2段：过于宽松，又引入噪声
- **±1段是最佳平衡**

### 长度兜底

- maxExcerptChars = 20,000字符 ≈ 13,000 tokens
- 3章教材 × 20,000 = 60,000字符 ≈ 40,000 tokens
- 加上8篇指南 ≈ 20,000字符 ≈ 13,000 tokens
- 总计 ≈ 53,000 tokens，在58,000预算内

如果教材章数多（>4章）：
```
perChapterBudget = (58000 - 指南tokens) / 教材章数 / 1.5 * 1.0
                   (字符=token×1.5，但中文1.5是宽松估计，×1.0安全)
实际: 按字符预算算
totalCharBudget = 87000
guideChars = 实际加载的指南总字符数
textbookCharBudget = 87000 - guideChars
perChapterChars = textbookCharBudget / 教材章数
maxExcerptChars = min(20000, perChapterChars)
```

### 回退逻辑

如果关键词几乎没命中（截取结果<500字符）：
- 说明这章教材虽被FTS/验证通过，但全文中与查询词的关联很弱
- 回退到取前5,000字符（至少有概述和开头内容）
- 这也比固定截断好——大部分情况下关键词会命中，回退是小概率

## 对方7项建议的最终决策

| 编号 | 建议项 | 决策 | 理由 |
|------|--------|------|------|
| 1 | 截断从7000→5000 | ❌ 不采纳固定数字 | 指南不截断，教材用关键词段落截取+动态预算分配 |
| 2 | 回退量从10→5 | ❌ 保持10条 | 提取agent有段落级过滤兜底，空结果比噪声更危险；加UI标注"低相关性文献" |
| 3 | 体裁判断交给LLM | ✅ 采纳 | type字段("推荐意见"|"知识要点"|"混合")，prompt给判断示例 |
| 4 | LLM失败回退 | ✅ 采纳 | 每步try-catch，详见下表 |
| 5 | 空文档检测 | ✅ 采纳 | 零成本防护 |
| 6 | 医生反馈字段 | ❌ 第二阶段 | 先跑通检索→验证→提取链路 |
| 7 | 提取agent输出使用/未使用原因 | ❌ 不采纳 | 增加复杂度，无临床操作价值 |

## LLM失败回退策略

| 步骤 | LLM调用 | 失败回退 |
|------|---------|---------|
| Step 1 查询扩展 | expandQueryWithLlm | 用原始diag.name搜FTS，不扩展 |
| Step 3 语义验证 | verifyResultsWithLlm | 不验证，直接用FTS排序取前10条 |
| Step 5 提取归纳 | extractAndSummarize | recommendations=[], summary="AI服务暂不可用，请直接查看参考原文" |
